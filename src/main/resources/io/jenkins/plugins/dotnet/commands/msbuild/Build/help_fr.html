La commande dotnet build génère le projet et ses dépendances dans un ensemble de fichiers binaires. Les binaires incluent le code du projet dans les fichiers de langue intermédiaire (IL) avec une extension .dll. Selon le type de projet et les paramètres, d’autres fichiers peuvent être inclus, tels que :
  1. Un exécutant qui peut être utilisé pour exécuter l’application, si le type de projet est un ciblage exécutable .NET Core 3.0 ou plus tard.
  2. Fichiers de symbole utilisés pour débogage avec une extension .pdb.
  3. Un fichier .deps.json, qui répertorie les dépendances de l’application ou de la bibliothèque.
  4. A .runtimeconfig.json, qui spécifie le temps d’exécution partagé et sa version pour une application.
  5. Autres bibliothèques dont dépend le projet (via des références de projet ou des références de paquets NuGet).
(voir aussi la Documentation de Microsoft)
